# InterviewLab — Roadmap v2.0

## 1. Резюме состояния

**Что хорошо.** Кодовая база зрелая и аккуратная. Дизайн-система Linear перенесена в токены один-в-один (`src/index.css`: dark-first палитра, один индиго-акцент `#5e6ad2`, hairline-границы, mono-цифры, `prefers-reduced-motion`). Cmd+K палитра, скелетоны, продуманные пустые состояния, прогресс-бары, aria-разметка и Radix focus-trap уже на месте. Бэкенд тоже солидный: тяжёлые вычисления на `spawn_blocking`, батч-LLM с ограниченной конкурентностью и волнами, чекпоинты, кооперативная отмена и watchdog для ASR и батч-CLI. Промпты имеют единый аналитический системный промпт и жёсткие серверные инварианты (id, тайминги, evidence-ссылки, goal_id).

**Главные боли.**
1. **Необратимость без защиты.** Удаление интервью (`interviews-tab.tsx:664-676`) и Re-transcribe/Re-clean/Re-diarize (`:584-662`) срабатывают с hover-кнопки в один клик, затирая транскрипт и ручные правки. Нет guard несохранённых изменений нигде, где есть ручное редактирование.
2. **Контекст LLM инжектится неравномерно.** Глоссарий доходит только до ASR/cleanup, но не до синтеза/summary/diff/chat — терминология в артефактах разъезжается. Синтез волны не видит предыдущие волны — знание не накапливается. Правки markdown-артефакта **никогда** не возвращаются в `findings_json`, поэтому diff и chat видят устаревшую машинную версию (`synthesis.rs:1951`, `:2079`).
3. **Тон и консистентность UI.** Смешение RU/EN на одном экране, кнопка с матом «Хуйня, переписывай» (`transcript-editor.tsx:551,564`), причём этот текст ещё и в `aria-label` — скринридер зачитает.
4. **Wide-режим не проработан.** Контент капается в `max-w-2xl/3xl` и прижат влево в full-width области — на 2560px+ половина экрана мёртвая. Во всём фронте ни одного `2xl:`.
5. **Артефакты заперты в приложении.** Нет ни одного экспорта/копирования синтеза, summary, diff.
6. **Навигация без контекста.** На cycle-detail нет имени цикла, вкладки живут в локальном state (нет deep-link), нет онбординга и хлебных крошек.

## 2. Сквозные темы (дедуплицировано)

**A. Обратимость и защита данных.** Удаление интервью, повторные деструктивные прогоны, отсутствие guard несохранённых правок, отсутствие undo. Паттерн `confirm()` уже есть в `guides.tsx:148` и `cycle-chat-panel.tsx:309` — применён непоследовательно. *(UX-эвристики)*

**B. Глоссарий не доходит до синтеза/summary/diff/chat.** Всплывает в трёх доменах сразу (промпты, артефакты, терминология). `render_for_prompt` уже готов и используется в `cleanup.rs`/`asr.rs` — нужно протянуть в `synthesis.rs build_extract_input/build_reduce_input` и `chat.rs build_context`. В `docs/transcription-terminology.md:67` это прямо помечено как несделанный follow-up.

**C. Двойное представление артефакта без обратной синхронизации.** `findings_json` (источник для diff/chat) и редактируемый `content_md` расходятся; правки пользователя невидимы для diff и чата. *(артефакты + промпты)*

**D. Терминология и тон UI.** RU/EN вперемешку, мат в подписи и aria-label, `cycle` vs `wave`, `Clean` vs «Хуйня, переписывай», `Diff` без пояснения, дублирующиеся словари статусов. *(UX, IA, консистентность, доступность)*

**E. Wide / адаптивная сетка отсутствует выше lg.** Узкие колонки прижаты влево, списки/таблицы без max-width, findings/diff одноколоночные, ни одного `2xl:`. *(wide-домен целиком)*

**F. Cmd+K и keyboard-first неполны.** Палитра знает 4 действия + переход к циклам; нет навигации в Guides/Products/интервью/вкладки; декоративный хинт «C»; нет реестра шорткатов и cheatsheet; редактор и волна недоступны с клавиатуры. *(доступность + IA)*

**G. Видимость статуса и ошибок.** Прогресс долгих операций в локальном state теряется при навигации; нет глобального индикатора фоновых задач; статус-дот циклов «врёт»; ошибки — сырой `String(e)` в исчезающем тосте. *(UX)*

**H. Производительность БД и процессов.** Нет индексов на горячих FK; N+1 в синтезе; чат-стрим без watchdog; нет `kill_on_drop`; двойное чтение WAV; нетроттленный прогресс ASR. *(бэкенд)*

**I. Дублирование UI-примитивов.** EmptyState, Pill/Tag, status-meta карты скопированы по 6+ мест; off-palette amber; битая CSS-переменная волны; мёртвый `App.css`. *(консистентность)*

**J. Трассируемость и экспорт артефактов.** segment_id как индекс ломается при склейке; markdown пишет «segment N» как мёртвый текст; diff без evidence; нет экспорта вообще. *(артефакты)*

## 4. Системные промпты и контекст LLM (детально)

Сейчас контекст инжектится неравномерно, `system`-поле для structured-задач — это просто ещё один ключ JSON, который сериализуется в один общий prompt+stdin (`adapter.rs:930-953`), настоящий system-канал (`--append-system-prompt-file`) используется только в chat. Глоссарий доходит только до ASR/cleanup. Правило языка вывода живёт только в `analysis_system_prompt`.

### Общая архитектура (применить ко всем шести стадиям)

**Единое системное ядро (один источник правды), переиспользуемое везде:**
- **Язык вывода** = язык интервью; не переводить термины.
- **Анти-галлюцинации**: не выдумывать имена/числа/цитаты; «inconclusive / not answered» лучше, чем додумывание.
- **Терминология**: использовать каноничные написания из глоссария в прозе и в цитатах.
- **Стиль артефактов**: нейтральный аналитический тон, без воды, единый формат цитат и чисел, без markdown-заголовков внутри строковых полей.

Вынести это в одну Rust-константу и подключать во всех стадиях поверх per-stage инструкций (убрать расходящиеся формулировки из `cleanup.rs guidelines_for` и `glossary.rs EXTRACT_GUIDELINES`). Перевести structured-задачи на `--append-system-prompt-file` (стабильный префикс = ядро + guidelines + glossary + style-guide; на stdin — только данные), что усилит соблюдение правил и откроет prompt-кэширование. Пока этого нет — хотя бы текстом маркировать в `render_prompt` границу `SYSTEM RULES (highest priority)` vs `DATA`.

### Что инжектить по шагам

- Опционально: **роли спикеров** в `InputSegment` (сейчас только speaker_label; `role_map_db` уже есть) — для пунктуации реплик.

**synthesis / extract (MAP)** (`synthesis.rs build_extract_input`, `~1048-1077`)

- Добавить: **глоссарий** (`render_for_prompt`) как авторитет написаний в quotes; **полный markdown гайда** (`effective_guide`) рядом со структурированными goals/questions — чтобы MAP видел намерение «зачем этот вопрос» и «какие целевые выводы ожидаются», а не только список.

Жёстко оговорить, что выводы опираются на доказательства текущей волны.

**diff** (`diff.rs build`, `:121-141`)

**glossary-suggest / extract** (`glossary.rs EXTRACT_GUIDELINES`)
- Подключить общий мини-блок правил (язык/анти-галлюцинации) вместо своей расходящейся формулировки.

### Закрепление словаря и единого стиля
- Глоссарий резолвится по `interview → cycle → product`; в синтезе он один на цикл (по product_id). Путь `glossary_for_interview_db` уже есть — переиспользовать.

- **Единая модель артефакта «structured = источник истины, markdown = проекция»**: read-only markdown или обратный парсинг `content_md` в `findings_json`, чтобы diff/chat видели правки пользователя. Снимает critical-расхождение.
- **Сквозная трассируемость**: стабильные неизменяемые id сегментов вместо индексов массива + кликабельные deep-link во всех артефактах (summary/synthesis/diff/chat), резолвящиеся даже после склейки/правок.
- **Слой экспорта отчёта цикла**: единый «Export report» (markdown → PDF/HTML), собирающий синтез + per-interview summary + diff + цитаты с таймкодами — финальный продукт исследования.
- **Кумулятивный синтез между волнами** + перевод structured-задач на настоящее system/user-разделение (`--append-system-prompt-file`) с единым системным ядром для всех шести стадий и prompt-кэшированием стабильного префикса.
- **Глобальный store фоновых задач** + сводный индикатор в шелле, переживающий навигацию (ASR/диаризация/очистка/синтез/дифф), с восстановлением реального прогресса.
- **Cmd+K как универсальный навигатор** + централизованный реестр шорткатов с cheatsheet по «?» + полностью клавиатурный редактор транскрипта (roving-навигация, доступная волна-слайдер).
- **Единая адаптивная сетка рабочей области** с явными правилами для `2xl` (1536px+), зафиксированная в `docs/design-direction.md`; findings/diff — многоколоночные на wide; Products/Guides — адаптивная 2–3-колоночная раскладка.
- **Базовый слой UI-примитивов** (EmptyState, Pill/Tag, расширенный StatusBadge, единый источник статус-вокабуляра) + CI-линт, запрещающий сырые палитровые классы и hex в `.tsx`.
- **Сквозная политика обратимости**: подтверждения для всех деструктивных действий, soft-delete с undo, guard несохранённых изменений, человекочитаемые ошибки с next-step вместо `String(e)`.
